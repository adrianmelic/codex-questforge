# Video Outline

Use this when recording a short real Codex session for X, LinkedIn, GitHub, or a project page. The goal is to show that Questforge is not a web game: it is a fantasy RPG played inside Codex, with local state and visual table aids.

## 45-Second Cut

1. Open on the cover image or the GitHub Pages preview for 2-3 seconds.
2. Cut to Codex and paste: `I want to play @questforge in English. Create a quick character and start.`
3. Show Questforge creating the character and first scene.
4. Show one player action in natural language.
5. Show the dice result and a visible state update: HP, XP, inventory, clue, clock, or checkpoint.
6. Show one generated scene image or the visual gallery.
7. End on the install prompt or repository URL.

## 90-Second Cut

1. Start with the thesis: "Codex is the game surface."
2. Show quick character creation.
3. Show the first scene and one open-ended player decision.
4. Show a rules lookup or dice roll.
5. Show `game-state.json` or a campaign memory file changing locally.
6. Show a generated image in the Codex conversation.
7. Show the visual gallery, tactical map, or 360 POV viewer.
8. Toggle ambience briefly if audio is enabled.
9. Close with the repo and the exact start prompt.

## Capture Notes

- Prefer a real Codex thread over the guided web sample.
- Keep the browser preview secondary; it is supporting material, not the product.
- Do not reveal private campaign folders, personal prompts, API keys, or unrelated local files.
- If recording in Spanish, add a short English caption so the concept stays clear to a broader audience.
- Avoid long waits in the edit: jump from action to result, then show the generated image or state update.
